### Integration Issues (Non-blocking)

1. **INT-01: Generator script paths** — `bindings/python/generator/generator.py` references `include/quiverdb/c/` but headers live at `include/quiver/c/`. Cannot regenerate `_declarations.py`. Non-blocking since `_c_api.py` uses hand-written declarations.

2. **INT-02: Package name documentation** — REQUIREMENTS.md and CLAUDE.md say package is named `quiver` but actual package is `quiverdb`. All runtime code and tests correctly use `quiverdb`.

3. **INT-03: import_csv signature** — `import_csv(table, path)` missing `group` parameter vs C API `quiver_database_import_csv(db, collection, group, path, opts)`. Non-blocking since stub raises before reaching C API.
