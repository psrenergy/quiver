## Tech Debt

12 items carried forward (3 reviewer-classified BLOCKER, 9 WARNING). All adjudicated during phase verification — no current behavior fails any Phase 1 truth or test.

### Phase 01-c-core

**Critical — deferred per human-decision (CR-01):**
- **CR-01** (`src/expr/nodes.cpp:132-151`): D-04 mirror-case gap — symmetric loop missing. When lhs has dim X as non-time and rhs has X as time, validation silently passes and rhs time semantics are dropped. No test currently exercises this. Decision: accept-and-defer (option a) to follow-up plan.

**Advisory — quality concerns surfacing in deferred milestone scope:**
- **CR-02** (`src/binary/binary_file.cpp:81-100`): `open_file('w',...)` leaks canonical path into `write_registry` on partial-construction failure. Self-corrects on process restart. No current test path.
- **CR-03** (`src/expr/nodes.cpp:110,127,141,148,157,168,177`): BinaryOpNode `runtime_error` messages don't match the 3 CLAUDE.md patterns (`"Cannot apply binary operation: ..."` is not a C++ method name). Will surface unchanged through deferred bindings.

**Quality / performance warnings:**
- **WR-01** (`src/expr/nodes.cpp:146-147`): D-04 over-restrictive — `parent_dimension_index` compared by raw operand index, not parent name.
- **WR-02** (`include/quiver/binary/binary_file.h:67`): `validate_file_is_open()` is dead code.
- **WR-03** (`src/binary/binary_file.cpp:394-464` vs `src/binary/iteration.cpp:21-91`): `dimension_sizes_at_values` duplicated verbatim (~70 lines).
- **WR-04** (`include/quiver/binary/binary_file.h:76-77`): `BinaryFile::dimension_sizes_at_values` and protected `next_dimensions` appear unused after the delegate refactor.
- **WR-05** (`src/binary/binary_file.cpp:269-372`): `validate_dimension_values` and `_indexed` duplicate ~50 lines of calendar-aware logic.
- **WR-06** (`src/expr/nodes.cpp:283-310`): `BinaryOpNode::compute_row` does O(n²) inner search per row for output dim mapping; pre-computed forward translation table would be O(n).
- **WR-07** (`src/expr/nodes.cpp:82-94`): `apply_op` returns `0.0` after exhaustive switch — silent fall-through risk if a `BinaryOp` variant is added later.
- **WR-08** (`include/quiver/expr/expression.h:23`): No test exercises implicit `BinaryFile → Expression` conversion in an operator argument.
- **WR-09** (`src/binary/iteration.cpp:104-140`): `next_dimensions` rebuilds `first_dimensions(meta)` per call to detect end-of-iteration; allocates a fresh vector and re-runs calendar-aware computation each step.

**Total:** 12 items across 1 phase.
