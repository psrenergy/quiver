### Phase 02-tech-debt: 5 minor review residuals (advisory, carry-forward)

The Phase 2 code review (`02-REVIEW.md`) surfaced 5 residual concerns within the changed files. None are regressions introduced by Phase 2; none block the phase goal; none fail any Phase 1 truth. They are listed for traceability:

- **REVIEW-WR-01** (`include/quiver/expr/node.h:60`): `FileNode::own_dims_buf_` declared but never used. Likely missed by Phase 2's WR-02..04 dead-code sweep. Minor cleanup candidate.
- **REVIEW-WR-02** (`src/expr/nodes.cpp:138-140`): `parent_name_of` returns empty-string sentinel that could theoretically collide with an empty dimension name. Not reachable through normal `validate()` paths.
- **REVIEW-WR-03** (`src/expr/nodes.cpp:138-140`): `parent_name_of` dereferences `m.dimensions[parent_idx]` without bounds check. UB only on invalid metadata that bypasses `validate()`.
- **REVIEW-WR-04** (`src/binary/binary_file.cpp:91-93`): Pre-existing TOML truncation on validation failure. CR-02 addressed registry leak, not filesystem atomicity. Out of Phase 2 scope.
- **REVIEW-WR-05** (`src/binary/binary_file.cpp:19,59,104`): Pre-existing single-thread policy. CR-02 late-insert widened the unguarded TOCTOU window. Safe under documented single-threaded contract.

**Total:** 5 residuals (3 are mitigated only by the project's documented "callers obey contracts" policy; 2 are pre-existing pre-Phase-2 concerns). Carry-forward candidates for a future cleanup pass if/when the project shifts to multi-threaded use or extends `BinaryMetadata::validate()`.
