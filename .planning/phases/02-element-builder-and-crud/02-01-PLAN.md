<objective>
Implement createElement and deleteElement on the Database class via prototype extension, with typed value marshaling from plain JavaScript objects to C API Element handles.

Purpose: Enable users to create elements with scalar (integer, float, string, null) and array (integer, float, string) values using a plain-object API (`db.createElement('Items', { label: 'Sword', damage: 42, tags: ['fire'] })`), and delete elements by ID. This is the JS/TS equivalent of Python's `create_element(**kwargs)` pattern.

Output: `types.ts` (reusable Value types), `create.ts` (createElement/deleteElement implementations), updated `loader.ts` (element + CRUD FFI symbols), updated `index.ts` (side-effect import), and `create.test.ts` (integration tests).

Implement using the prototype extension pattern with `declare module`:


1. `declare module "./database"` block adding `createElement(collection: string, data: ElementData): number` and `deleteElement(collection: string, id: number): void` to the Database interface.

2. `setElementField(lib, elemPtr, name, value)` internal function:
   - `null` -> `check(lib.quiver_element_set_null(elemPtr, cName))`
   - `bigint` -> `check(lib.quiver_element_set_integer(elemPtr, cName, value))`
   - `number` + `Number.isInteger(value)` -> `check(lib.quiver_element_set_integer(elemPtr, cName, value))`
   - `number` + not integer -> `check(lib.quiver_element_set_float(elemPtr, cName, value))`
   - `string` -> `check(lib.quiver_element_set_string(elemPtr, cName, toCString(value)))`
   - `Array.isArray(value)` -> call `setElementArray(lib, elemPtr, name, value)`
   - Otherwise -> `throw new QuiverError("Unsupported value type for '${name}': ${typeof value}")`

3. `setElementArray(lib, elemPtr, name, values)` internal function:
   - Empty array (length === 0): `check(lib.quiver_element_set_array_integer(elemPtr, cName, 0, 0))` -- pass null pointer with count 0
   - First element is bigint: `BigInt64Array.from(values, v => BigInt(v))`, pass with `ptr(arr)` and `values.length`
   - First element is number:
     - Check ALL values: `values.every(v => Number.isInteger(v))` -- if true, integer array path (convert to BigInt64Array); if false, float array path (Float64Array)
     - Per locked decision: "number[] where ALL values pass Number.isInteger() -> integer array; ANY decimal -> float array"
   - First element is string: Build pointer table using `BigInt64Array` holding `BigInt(ptr(buffer))` for each `toCString(s)`. Keep buffer references alive in local array scope.
   - Otherwise: `throw new QuiverError("Unsupported array element type for '${name}': ${typeof values[0]}")`

4. `Database.prototype.createElement` implementation:
   - `const lib = getSymbols();`
   - `this._handle` (triggers ensureOpen)
   - Create element handle: `const outElem = allocPointerOut(); check(lib.quiver_element_create(ptr(outElem))); const elemPtr = readPointerOut(outElem);`
   - try block: iterate `Object.entries(data)`, skip `undefined` values, call `setElementField` for each
   - Create in database: `const outId = new BigInt64Array(1); check(lib.quiver_database_create_element(this._handle, toCString(collection), elemPtr, ptr(outId)));`
   - Return `Number(outId[0])`
   - finally block: `lib.quiver_element_destroy(elemPtr)`

5. `Database.prototype.deleteElement` implementation:
   - `const lib = getSymbols();`
   - `check(lib.quiver_database_delete_element(this._handle, toCString(collection), id))`
