
   c. Implement `marshalParams(params: QueryParam[])` as a module-private function:
      - Create `Int32Array(n)` for types and `BigInt64Array(n)` for values (pointer table)
      - Create `keepalive: unknown[]` array to prevent GC of intermediate buffers
      - For each param: null -> type=4, values[i]=0n; integer (Number.isInteger) -> type=0, BigInt64Array([BigInt(p)]) stored in keepalive, values[i]=BigInt(ptr(buf)); float -> type=1, Float64Array([p]) stored in keepalive; string -> type=2, toCString(p) stored in keepalive
      - Return `{ types, values, _keepalive: keepalive }`
      - Use constants: DATA_TYPE_INTEGER=0, DATA_TYPE_FLOAT=1, DATA_TYPE_STRING=2, DATA_TYPE_NULL=4

   d. Implement `Database.prototype.queryString`:
      - Allocate outValue with `allocPointerOut()` (char** out-param) and outHasValue with `new Int32Array(1)`
      - If params present and length > 0: call `quiver_database_query_string_params` with marshaled types/values/count
      - Else: call `quiver_database_query_string` with just sql/outValue/outHasValue
      - If outHasValue[0] === 0 return null
      - Else: readPointerOut(outValue) -> CString -> toString(), then free_string, return result

      - Allocate outValue with `new Float64Array(1)` (double* out-param) and outHasValue with `new Int32Array(1)`
      - Same dispatch pattern as queryString
      - If has value: return `outValue[0]`
