Key field rationale:
- `name` and `version` carried over from package.json (name: "quiverdb", version: "0.6.3")
- `exports.".": "./src/index.ts"` -- Deno uses TypeScript source directly, no dist/ build step
- `tasks.test` uses `--allow-ffi --allow-read --allow-write --allow-env` (tests need FFI for dlopen, read for schemas, write for temp DBs, env for PATH)
- `tasks.lint` and `tasks.format` keep biome commands (biome.json stays as-is, per RESEARCH recommendation)
- `fmt` section mirrors biome.json settings (indentWidth: 2, lineWidth: 100, semiColons: true, singleQuote: false) for deno fmt compatibility
- `nodeModulesDir`: `"none"` (string, not boolean -- boolean form deprecated in Deno 2.x)
- No `imports` field needed: loader.ts uses inline `jsr:@std/path` specifier (only one file uses it, per RESEARCH recommendation to keep inline)
- No `compilerOptions` field: Deno defaults (strict: true, module: nodenext, target: esnext) match the current tsconfig.json settings
