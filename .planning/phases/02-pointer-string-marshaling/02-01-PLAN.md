4. **allocUint64Out** (per D-01, D-03): Returns `Allocation`. Create `Uint8Array(8)`. Return `{ ptr: Deno.UnsafePointer.of(buf)!, buf }`.

5. **readUint64Out** (per MARSH-01): Takes `Allocation`, returns `number`. Create `DataView(alloc.buf.buffer)`, read `Number(getBigUint64(0, true))`.

6. **decodeStringFromBuf** (per D-08, MARSH-03): Takes `Allocation`, returns `string`. Call `readPtrOut(alloc)` to get the char* pointer. If null, return `""`. Otherwise return `new Deno.UnsafePointerView(ptr).getCString()`.

7. **toCString** (per D-04, D-08, MARSH-03): Takes `string`, returns `Allocation`. Use `encoder.encode(str + "\0")` to get null-terminated UTF-8 Uint8Array. Return `{ ptr: Deno.UnsafePointer.of(encoded)!, buf: encoded }`.

9. **allocNativeFloat64** (per D-01, D-02, MARSH-04): Takes `number[]`, returns `Allocation`. Create `Uint8Array(values.length * 8)`, use `DataView` to write `setFloat64(i * 8, values[i], true)` for each value. Return `{ ptr: Deno.UnsafePointer.of(buf)!, buf }`.

10. **allocNativePtrTable** (per D-01, MARSH-04): Takes `Deno.PointerValue[]`, returns `Allocation`. Create `Uint8Array(ptrs.length * 8)`, use `DataView`. For each pointer, get address via `ptrs[i] ? Deno.UnsafePointer.value(ptrs[i]!) : 0n`, write with `setBigUint64(i * 8, addr, true)`. Return `{ ptr: Deno.UnsafePointer.of(buf)!, buf }`.

11. **allocNativeString** (per D-01, D-08, MARSH-04): Takes `string`, returns `Allocation`. Use `encoder.encode(str + "\0")`. Return `{ ptr: Deno.UnsafePointer.of(buf)!, buf }`. (Same as toCString -- both produce null-terminated C strings, but allocNativeString is used in contexts where the string becomes part of a pointer table, while toCString is used for direct string parameters.)

12. **allocNativeStringArray** (per D-01, D-03, MARSH-04): Takes `string[]`, returns `{ table: Allocation; keepalive: Allocation[] }`. Map strings through `allocNativeString`, collect pointers, pass to `allocNativePtrTable`. Return `{ table, keepalive: strAllocs }`.

13. **nativeAddress**: Takes `Deno.PointerValue`, returns `bigint`. Return `ptr ? Deno.UnsafePointer.value(ptr) : 0n`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JS -> Native (FFI call) | Marshaled buffers cross from JS heap to native C code via Deno.dlopen |
| Native -> JS (pointer read) | Native pointers decoded back to JS values via UnsafePointerView |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | toCString, allocNativeString | mitigate | Always append "\0" null terminator before encoding with TextEncoder (prevents C from reading past buffer into adjacent memory). Verified in every string function -- `encoder.encode(str + "\0")` |
| T-02-02 | Denial of Service | allocPtrOut, allocNativeInt64, all alloc functions | mitigate | Return `{ ptr, buf }` Allocation from every function so callers hold buffer references (prevents GC from collecting buffers while native code holds dangling pointers). Per D-03 and D-06 |
| T-02-03 | Denial of Service | readPtrOut | mitigate | Check for `raw === 0n` before `Deno.UnsafePointer.create(raw)` to avoid null dereference when C function writes NULL to out-parameter. Returns `null` PointerValue instead |
| T-02-04 | Information Disclosure | makeDefaultOptions | accept | Struct layout (8 bytes, 2x int32) is fixed by C header. Wrong offsets would cause misinterpretation. Low risk: values are read-only flag and log level, no secrets. Layout verified against include/quiver/c/options.h |
| T-02-05 | Tampering | DataView byte order | mitigate | All DataView get/set calls pass `true` for littleEndian parameter. Matches x86/ARM native byte order. Per D-02, verified in research Pitfall 3 |
</threat_model>
