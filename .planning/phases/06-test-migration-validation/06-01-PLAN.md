  <action>
For each of the 3 files, apply this exact transformation:

**1. Replace the vitest import line:**
Remove: `import { describe, expect, test } from "vitest";`

**2. Add Deno std/assert imports:**
Add as the first line: `import { assert, assertEquals, assertGreater, assertInstanceOf, assertNotEquals, assertThrows } from "jsr:@std/assert";`
Only import the specific assertions actually used in that file. The mapping is:
- `expect(x).toBe(y)` -> `assertEquals(x, y)`
- `expect(x).toEqual(y)` -> `assertEquals(x, y)`
- `expect(x).toBeNull()` -> `assertEquals(x, null)`
- `expect(x).toBeDefined()` -> `assertNotEquals(x, undefined)`
- `expect(x).toBeInstanceOf(Y)` -> `assertInstanceOf(x, Y)`
- `expect(x).toContain(y)` for arrays -> `assert(x.includes(y))`
- `expect(x).toHaveLength(n)` -> `assertEquals(x.length, n)`
- `expect(x).toBeGreaterThan(y)` -> `assertGreater(x, y)`
- `expect(fn).toThrow(Type)` -> `assertThrows(fn, Type)`
- `expect(typeof x).toBe("number")` -> `assertEquals(typeof x, "number")`
- `expect(typeof x).toBe("string")` -> `assertEquals(typeof x, "string")`

**3. Replace node:path dirname pattern:**
Remove these 3 lines:
```
import { dirname } from "node:path";
import { fileURLToPath } from "node:url";
const __dirname = dirname(fileURLToPath(import.meta.url));
```
Replace with: `const __dirname = import.meta.dirname!;`

**4. Replace node:path join:**
Remove: `import { join } from "node:path";`
Replace with: `import { join } from "jsr:@std/path";`

**5. Fix src imports to use .ts extension:**
Change: `from "../src/index"` -> `from "../src/index.ts"`
Change: `from "../src/types"` -> `from "../src/types.ts"` (if present)

**6. Convert describe/test to Deno.test/t.step:**
Each vitest `describe("Name", () => { ... })` block becomes:
```typescript
Deno.test("Name", async (t) => {
  await t.step("test name", () => {
    // test body
  });
});
```

Each vitest `test("name", () => { body })` inside a describe becomes `await t.step("name", () => { body })`.

**7. Convert all assertions per the mapping above.**

**Specific per-file notes:**

**create.test.ts:**
- Uses `QuiverError`, `ElementData`, `Value` types -- keep these imports, fix extension
- Has nested `try { expect(...).toThrow(QuiverError) } catch` patterns -- replace with `assertThrows(() => { ... }, QuiverError)`
- The pattern `expect((e as QuiverError).message).toContain("some string")` should become: inside a try/catch, use `assertStringIncludes((e as QuiverError).message, "some string")` -- add `assertStringIncludes` to the import
- `expect(x).toBeGreaterThan(0)` -> `assertGreater(x, 0)`

**read.test.ts:**
- Heavy use of `toEqual` for array comparisons -> `assertEquals`
- `toHaveLength(n)` -> `assertEquals(x.length, n)`
- `toContain(id1)` for array membership -> `assert(ids.includes(id1))`
- `toBeNull()` -> `assertEquals(value, null)`

**update.test.ts:**
- Uses `toBeNull()` -> `assertEquals(value, null)`
- Uses `toBe(99)` etc -> `assertEquals(value, 99)`
