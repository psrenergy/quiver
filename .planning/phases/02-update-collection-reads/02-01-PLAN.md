3. In `bindings/js/src/read.ts`, add 12 new methods to the `declare module "./database"` block and implement via prototype assignment:

   **Bulk vector reads (return `number[][]` or `string[][]`):**
   - `readVectorIntegers(collection, attribute)`: Allocate `outVectors` (allocPointerOut for T***), `outSizes` (allocPointerOut for size_t**), `outCount` (BigUint64Array(1) for size_t*). Call FFI. Dereference: `vectorsPtr = readPointerOut(outVectors)`, `sizesPtr = readPointerOut(outSizes)`. Read sizes via `toArrayBuffer(sizesPtr, 0, count * 8)` into `BigUint64Array`. For each element i: if size[i] === 0, push empty array; else `readPtrAt(vectorsPtr, i * 8)` to get inner int64_t*, `toArrayBuffer` to read, convert BigInt64Array to number[]. Free with `quiver_database_free_integer_vectors(vectorsPtr, sizesPtr, count)`.
   - `readVectorFloats(collection, attribute)`: Same double-deref pattern but inner arrays use `Float64Array`. Free with `free_float_vectors`.
   - `readVectorStrings(collection, attribute)`: Triple dereference -- outer gives char***, for each element `readPtrAt` to get char**, for each string in that array `readPtrAt` to get char*, wrap in `new CString()`. Free with `free_string_vectors`.

   **Bulk set reads (identical implementation pattern to vector bulk, just different FFI function names):**
   - `readSetIntegers(collection, attribute)`: Same as readVectorIntegers but calls `quiver_database_read_set_integers` and frees with `quiver_database_free_integer_vectors` (sets reuse vector free functions).
   - `readSetFloats(collection, attribute)`: Same pattern, different FFI names.
   - `readSetStrings(collection, attribute)`: Same pattern, different FFI names.

   **By-id vector reads (return `number[]` or `string[]` -- flat arrays, same pattern as existing readScalarIntegers):**
   - `readVectorIntegersById(collection, attribute, id)`: Allocate `outValues` (allocPointerOut), `outCount` (BigUint64Array(1)). Call `quiver_database_read_vector_integers_by_id(handle, collection, attribute, id, ptr(outValues), ptr(outCount))`. Dereference, read BigInt64Array, convert to number[]. Free with `quiver_database_free_integer_array`.
   - `readVectorFloatsById(collection, attribute, id)`: Same but Float64Array. Free with `free_float_array`.
   - `readVectorStringsById(collection, attribute, id)`: Deref to get char**, readPtrAt for each string, CString. Free with `free_string_array(arrPtr, count)`.

   **By-id set reads (identical to vector by-id, different FFI function names):**
   - `readSetIntegersById(collection, attribute, id)`: Same as readVectorIntegersById but calls `quiver_database_read_set_integers_by_id`. Free with `free_integer_array`.
   - `readSetFloatsById(collection, attribute, id)`: Same pattern.
   - `readSetStringsById(collection, attribute, id)`: Same pattern.

   **CRITICAL pitfalls to handle:**
   - For bulk reads: Always check `count === 0` and return `[]` early before any pointer dereference
   - For inner arrays: Check `size === 0` before calling `readPtrAt` (C API returns nullptr for empty inner arrays)
   - String bulk reads have one MORE level of pointer indirection than integer/float (char**** vs int64_t***)
   - All by-id reads returning empty data: if `count === 0` return `[]` (element may have no vector/set entries)
