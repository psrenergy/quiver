1. In `bindings/js/src/loader.ts`, add 2 FFI symbol declarations:
   - `quiver_database_export_csv`: args `[ptr, ptr, ptr, ptr, ptr]`, returns `i32`
   - `quiver_database_import_csv`: args `[ptr, ptr, ptr, ptr, ptr]`, returns `i32`

2. Create `bindings/js/src/csv.ts` with module augmentation on Database:
   - Define a `CsvOptions` interface:
     ```typescript
     export interface CsvOptions {
       dateTimeFormat?: string;
       enumLabels?: Record<string, Record<string, Record<string, number>>>;
     }
     ```
   - Implement a `buildCsvOptionsBuffer` function that constructs the 56-byte `quiver_csv_options_t` struct in a `Uint8Array`:
     - The struct has 7 fields, each 8 bytes on 64-bit (total 56 bytes).
     - For the default case (no options or empty options): set `date_time_format` to a pointer to an empty C string, all enum pointers to 0 (NULL), and `enum_group_count` to 0.
     - For the enum labels case: flatten the nested `Record<string, Record<string, Record<string, number>>>` into grouped parallel arrays (same logic as Python's `_marshal_csv_options`):
       - For each `(attribute, locale)` pair, collect group attribute names, locale names, entry counts, all labels, all values.
       - Allocate native buffers for each array (using `Buffer.from` for strings and typed arrays for counts/values).
       - Write pointers into the struct at correct offsets using `BigInt64Array` or `DataView`.
     - Use a `keepalive` array to hold references to all allocated buffers so GC does not collect them during the FFI call.
     - Since bun:ffi `ptr()` returns a `number`, use `BigInt(ptr(buffer))` when writing pointer values into the struct via `DataView.setBigInt64()`.
   - Implement `exportCsv(collection: string, group: string, filePath: string, options?: CsvOptions): void`:
     - Call `this._handle` to ensure database is open.
     - Build the CSV options buffer using `buildCsvOptionsBuffer`.
     - Call `lib.quiver_database_export_csv(this._handle, toCString(collection), toCString(group), toCString(filePath), ptr(optionsBuffer))`.
     - Wrap in `check()` for error handling.
   - Implement `importCsv(collection: string, group: string, filePath: string, options?: CsvOptions): void`:
     - Same pattern as exportCsv but calls `lib.quiver_database_import_csv`.
   - IMPORTANT: Use little-endian byte order for all pointer writes (`DataView.setBigInt64(offset, value, true)`) since the target is x86-64.

3. In `bindings/js/src/index.ts`:
   - Add `import "./csv";` alongside existing module imports.
   - Add `export type { CsvOptions } from "./csv";` to the type exports.
