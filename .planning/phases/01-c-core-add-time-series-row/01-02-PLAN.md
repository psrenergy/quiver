    **TEST 8: `AddTimeSeriesRowTransactionRollback`** (CORE-13, success criterion 3)
    Schema `collections.sql`. Create Configuration + one Collection element. Wrap an `add_time_series_row` call inside an explicit transaction and roll back:
    ```cpp
    db.begin_transaction();
    EXPECT_TRUE(db.in_transaction());
    db.add_time_series_row("Collection", "data", id, {{"date_time", std::string("2024-01-01")}, {"value", 1.0}});
    EXPECT_TRUE(db.in_transaction());  // still inside outer txn — nest-aware
    db.rollback();
    EXPECT_FALSE(db.in_transaction());

    auto rows = db.read_time_series_group("Collection", "data", id);
    EXPECT_TRUE(rows.empty());  // rolled back, nothing persisted
    ```

    **TEST 9: `AddTimeSeriesRowTransactionCommitAndStandalone`** (CORE-13, success criterion 3)
    Schema `collections.sql`. Create Configuration + one Collection element. Two phases:

    Phase A (explicit transaction commit, multiple writes batch together):
    ```cpp
    db.begin_transaction();
    db.add_time_series_row("Collection", "data", id, {{"date_time", std::string("2024-01-01")}, {"value", 1.0}});
    db.add_time_series_row("Collection", "data", id, {{"date_time", std::string("2024-01-02")}, {"value", 2.0}});
    EXPECT_TRUE(db.in_transaction());
    db.commit();
    EXPECT_FALSE(db.in_transaction());
    auto rows1 = db.read_time_series_group("Collection", "data", id);
    EXPECT_EQ(rows1.size(), 2);
    ```

    Phase B (standalone — no explicit txn, commits automatically):
    ```cpp
    EXPECT_FALSE(db.in_transaction());
    db.add_time_series_row("Collection", "data", id, {{"date_time", std::string("2024-01-03")}, {"value", 3.0}});
    EXPECT_FALSE(db.in_transaction());  // returned to autocommit after the call
    auto rows2 = db.read_time_series_group("Collection", "data", id);
    EXPECT_EQ(rows2.size(), 3);  // includes the prior 2 + this 1
    ```

    These two phases together prove the `TransactionGuard` nest-detection works both ways.
