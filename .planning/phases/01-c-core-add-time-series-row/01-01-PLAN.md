<objective>
Add the single-row append/upsert primitive `Database::add_time_series_row(collection, group, id, row)` to the C++ core, mirroring the structure of `update_time_series_group` (in `src/database_time_series.cpp:114-205`) but emitting `INSERT OR REPLACE INTO` against the time series table so the time series PK (`id` + every dimension column from the schema) yields upsert semantics. Validation, error-message prefix, and `TransactionGuard` usage match the sibling API exactly.

Purpose: Closes the gap between `read_time_series_row` (already exists) and `update_time_series_group` (replace-all) by giving callers a way to insert OR overwrite a single row without wiping the rest of the element's series. Also unblocks Phase 2 (C API wrapper) and Phase 3 (bindings) by establishing the canonical C++ signature.

Output: Header declaration, source implementation, and a new test schema with a composite-PK time series table (`date_time + block`) that Plan 02 will exercise.
</objective>
